# 📝 AI Word 智能排版助手 (AI Word Formatter)

本工具利用大语言模型（LLM）的深度文本理解能力，能够根据纯口语化的自然语言指令，一键自动对 Word 文档进行段落识别与专业的学术级格式排版，并支持全自动生成并渲染标准的 Word 域目录与真实页码。

---

## 🌟 零基础保姆级使用教程

### 第一步：安装 Python 并搭建基础环境
1. **安装 Python：** 
   请前往 [Python 中国镜像 / 官网](https://www.python.org/downloads/) 下载 Python (推荐版本 3.10 或以上)。安装时，**请务必勾选 "Add Python to PATH"**。
2. **下载本项目并进入目录：**
   将本项目下载后解压，在文件夹空白处点击右键，在弹出的菜单中选择 **“在终端中打开”**（或直接在上方地址栏输入 `cmd` 后按回车）。

### 第二步：创建虚拟环境并安装依赖
在刚打开的终端（黑色窗口环境）中，依次复制并回车运行以下命令：
```cmd
# 1. 创建名为 venv 的专属虚拟环境（隔离干净）
python -m venv venv

# 2. 激活虚拟环境 (Windows系统使用下面这行代码)
venv\Scripts\activate
# (如果是 Mac 或 Linux，请使用： source venv/bin/activate )

# 3. 安装项目运行所需的一键依赖包
pip install -r requirements.txt
```

### 第三步：获取并配置人工智能 API Key
本项目依赖于大模型引擎来分析你的内容，你可以去申请便宜、甚至首月免费的国产大模型 API 密钥（如深度求索 DeepSeek、字节豆包 Doubao 或月之暗面 Kimi）。
1. 请前往对应大模型的开放平台，注册并生成一串 `API_KEY` （通常以 `sk-` 开头）。
2. 在本项目根目录下，找到 `.env.example` 文件。**将它的名字改成 `.env`**。
3. 用记事本（或任意代码编辑器）打开改名后的 `.env` 文件，按如下格式进行填写（注意修改为你实际的 `api_key` 和对应的平台网址）：
```ini
# 以 DeepSeek 举例：
OPENAI_API_KEY=sk-*************你的Key*************
OPENAI_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

### 第四步：启动你的排版助手！
保证上面终端你的环境已经是激活状态（命令行开头有 `(venv)` 标志），即可直接输入并运行：
```cmd
python web_ui.py
```
终端会输出一行网址：`http://127.0.0.1:7860`。同时浏览器会自动打开它！
**进入页面后：上传你的乱糟糟的纯文本 Word，点击按钮，见证奇迹吧！** 🎉

## 🔧 核心功能体系

* **自然语言直接排版：** 不必记忆各种复杂的模板，你可以像说话一样告诉 AI ：“正文部分统一帮我改成宋体小四，1.5倍行距首行空两格；大标题全部居中加粗换成黑体”，它都能理解执行并落地修改。
* **基于特征规则底层的目录渲染：** 支持从一堆文字里智能抓取所有章节逻辑并分层，独立起开一页自动绘制成具有真正“域规则”和真实计算出页码的标准 Word 目录页（支持Word原生点击跳转及自动首行更新）。
* **自动页码与独立分节：** 完全隔离了标题目录与正文正本部分。让页脚能够清静并且从正文第1页开始强制输出符合学术发表逻辑的下居中阿拉伯数字。
* **智能断代图表、引用识别**：深度拆解学术论文里的结构，它甚至能够单独识别那句话是“图表的标注”，或者是“参考文献”，并为其单独赋子样式策略（支持如文献标号自动升维）。
* **完美学术级悬挂缩进**：对于大段的有序列表、带编号标题（1.1.1），系统从底层干预 Word 跳崖式制表符，全部替换为最严苛的学术极悬挂缩进。并对正文引文进行严格的上标及 Times New Roman 转换。

## 📁 核心架构层级

```text
wordcode/
├── assets/                  # 存放项目截图或文档相关的静态资源
├── docs/                    # 额外设计方案文档
├── src/                     # 核心引擎模块
│   ├── ai_parser.py         # AI 结构理解侧 (调用大模型解析、分类文档块)
│   └── formatter.py         # 排版引擎物理侧 (操作底层 OXML 与调用 COM 接口对齐样式应用)
├── tests/                   # 独立单元测试目录
├── web_ui.py                # Gradio Web 端交互入口 (极简前端，全功能控制)
├── main.py                  # CLI 命令行静默调用入口 (适合后台批处理排版)
├── .env.example             # API 配置环境示例表
├── requirements.txt         # 包管理配置
├── README.md                # 使用说明
└── .gitignore               # Git版本控制忽略项
```

## ⚙️ 进阶（Cli命令行）自动化调用

如果你希望把本工具集成在流水线爬虫或其他工程里大规模跑，我们提供了强力的 CLI 接口：

```bash
# 采用默认的最严苛学术排版模板，并打开 `--toc` 全自动抓取并重排目录
python main.py "./示例草稿.docx" "请用默认样式排版" --output "./输出结果.docx" --toc
```
